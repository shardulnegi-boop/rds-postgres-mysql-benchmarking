Interpretation Guide

  • CPU > 80% — Engine is compute-bound. Consider larger instance or connection pooling.
  • FreeableMemory < 10% — Buffer pool pressure. Rows spilling to disk, expect latency spike.
  • WriteLatency > 5ms — Storage layer struggling. Aurora storage is distributed; high latency means quorum writes are slow.
  • DiskQueueDepth > 10 — I/O requests queuing up. Storage can't keep pace with write rate.
  • BufferCacheHitRatio < 99% — Reads hitting disk instead of cache. Increase buffer pool or reduce working set.
  • ReplicaLag > 100ms — Readers falling behind writer. At sustained lag, read-after-write consistency breaks.
  • Deadlocks / Lock Waits — Concurrent writes conflicting. Review schema, indexing, and batch sizes.
  • PASS = achieved >= 80% of target | DEGRADED = 50-80% | FAIL = below 50%
